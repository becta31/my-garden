# Политика безопасности

## Сообщения об уязвимостях
Если вы обнаружили проблему безопасности, пожалуйста, **не создавайте публичный Issue**. Напишите владельцу репозитория напрямую или через личные сообщения.

## Защита данных
Проект использует секреты GitHub Actions для хранения API-ключей. 

**Критически важно:**
1. Никогда не сохраняйте файл `.env` в репозиторий.
2. Файл `.env` должен быть всегда добавлен в `.gitignore`.
3. Если вы случайно "засветили" свой `TELEGRAM_TOKEN` или `GEMINI_API_KEY` в истории коммитов — **немедленно перевыпустите ключ** в соответствующем сервисе, так как старый ключ считается скомпрометированным.

## Обновления
Мы рекомендуем регулярно обновлять зависимости из `requirements.txt`, чтобы закрывать дыры в безопасности используемых библиотек.
