🌿 Smart Garden Bot (AI-Powered)
Python 3.12License: MIT

Умная система уведомлений для домашнего сада. Бот не просто напоминает о поливе, но и анализирует погоду на улице, ведет статистику и дает советы от ИИ.

✨ Основные функции
🧠 Память бота: Бот запоминает, кого полил, и не спамит каждый день. Если вы полили растение (или бот напомнил), оно исчезнет из списка до следующего раза.
🤖 ИИ-советы: Подключение к Google Gemini для генерации полезных советов по уходу в зависимости от погоды и сезона.
🌡 Учет погоды: Интеграция с OpenWeatherMap. Бот знает температуру и влажность, давая рекомендации (например, "Холодно, поливай теплой водой").
📅 Умный график: Автоматический расчет полива и подкормок на основе базы данных plants.json.
🚀 Бесплатно и без сервера: Работает автоматически через GitHub Actions. Вам не нужен компьютер или сервер 24/7.
📱 Удобное управление: Интеграция с Telegram (напоминания и кнопки подтверждения).
🛠 Технологический стек
Python 3.12 — основная логика.
GitHub Actions — планировщик задач и хранение "памяти".
Telegram Bot API — интерфейс.
Google Gemini API — генерация советов.
JSON — база данных растений и истории.
📂 Структура проекта
send_tasks.py — основной скрипт: проверяет погоду, считает дни, спрашивает ИИ, отправляет сообщение в Telegram.
process_updates.py — обрабатывает нажатия кнопок в Telegram (например, "Сделано вручную").
plants.json — единый источник правды. Здесь хранятся данные о растениях (название, частота полива, стадия, удобрения).
history.json — файл "памяти". Хранит даты последних поливов.
last_weather.json — кэш погоды для отслеживания резких изменений температуры.
telegram_state.json — технический файл для работы Telegram бота.
🚀 Настройка для себя
Сделайте Fork репозитория.
Настройте Secrets (Settings > Secrets and variables > Actions):
TELEGRAM_TOKEN: Токен от @BotFather.
TELEGRAM_CHAT_ID: Ваш ID (можно узнать у @userinfobot).
OPENWEATHER_API_KEY: Ключ от OpenWeather.
GEMINI_API_KEY: Ключ от Google AI Studio (для советов ИИ).
CITY_NAME: Ваш город (например, Moscow).
Важно: Настройте права записи
Перейдите в Settings > Actions > General.
В разделе Workflow permissions выберите Read and write permissions.
Сохраните изменения. Без этого бот не сможет сохранять историю полива!
Настройте растения
Отредактируйте файл plants.json. Измените названия, частоту полива (waterFreq) и режим (stage).
📝 Как это работает
Каждый день (по расписанию cron) GitHub Actions запускает скрипт.
Скрипт скачивает погоду и читает файл history.json.
Для каждого растения проверяется: Текущая дата - Дата полива >= Частота полива.
Если условие верно — растение попадает в список.
ИИ (Gemini) генерирует совет на основе погоды и списка.
Сообщение отправляется в Telegram.
Важно: Скрипт делает git commit и push, обновляя history.json. Теперь растение считается политым.
🔧 Troubleshooting
Бот пишет одно и то же каждый день?Проверьте права записи (пункт 3 в настройке). Убедитесь, что в истории коммитов появляются обновления history.json.

Ошибка list object has no attribute get?Убедитесь, что history.json содержит валидный JSON. Если файл поврежден, просто удалите его содержимое (оставьте {}) — бот восстановит его при следующем запуске.
