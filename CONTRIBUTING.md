# Правила участия в разработке 🌿

Мы рады любой помощи! Вот как вы можете внести свой вклад:

### Как предложить изменения
1. Сделайте **Fork** репозитория.
2. Создайте свою ветку: `git checkout -b feature/новая-фича`.
3. Закоммитьте изменения: `git commit -m 'Добавлена новая функция'`.
4. Отправьте ветку в свой репозиторий: `git push origin feature/новая-фича`.
5. Создайте **Pull Request**.

### Требования к коду
* Код должен соответствовать стилю PEP 8.
* Если вы добавляете новую библиотеку, не забудьте обновить `requirements.txt`.
* Все текстовые сообщения в боте должны быть на русском языке (или поддерживать локализацию).

### Сообщения об ошибках
Если вы нашли баг, создайте **Issue**, подробно описав проблему и шаги для её воспроизведения.
